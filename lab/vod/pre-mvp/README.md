# 観点指向設計 (VOD) の最小構造

## 🎓 目的  
観点指向設計 (VOD) が、  
**変数固定のハードコーディングから疎結合の柔軟性を獲得し、  
単一世界から動的世界を表現できるようになるまでの進化過程**  
を示す。

その最初の段階である **pre‑MVP** では、  
VOD の本質である 3 層構造：

- **viewpoint**（意味の源泉）  
- **morphism**（一方向の関係）  
- **carrier**（意味ゼロの受像面）  

だけを扱う。

この 3 層は、後にどれだけ機能が拡張されても崩れない  
**VOD の OS カーネル** となる。

------------------------------------

# 00-pre-MVP  
## 🎓 概要  
pre‑MVP で最も重要なのは、  
**viewpoint → morphism → carrier**  
という **一方向の流れ**。

この流れを固定することで、構造的に以下が保証される：

- **処理の逆流の防止**（一方向性）  
- **循環参照の防止**（非再帰構造）  
- **意味の混濁の防止**（意味の所有権の分離）  

この段階では、まだ意味構造はほとんど存在しない。  
ただし **流れの純度だけは完全に確保する**。

------------------------------------

# 📕 用語

## 🎞️ viewpoint  
**意味の源泉を一意的に保持する場所。**  
写真で言えば「ネガ」、レコードで言えば「原盤」。

### 役割  
- 世界の一部をどう切り取るかという **局所世界 (local world)** を形成する  
- 意味の所有権 (ownership) を持つ唯一の場所  
- carrier に意味を置かないことで **意味の混濁を構造的に防止**  

### viewpoint が持つもの  
- **意味の源泉**  
- **局所世界の構造**  

### viewpoint が持たないもの  
- 変数  
- 内部状態  
- carrier の情報  

------------------------------------

## 🖇️ morphism  
**接続元と接続先の「関係」を表すもの。**  
圏論でいう「射」。  
写真で言えば「現像」。

### 役割  
- viewpoint の意味を carrier に **一方向に** 投影する  
- viewpoint → viewpoint の変換も扱える（後の拡張）  
- 逆流しないことで **循環参照を構造的に防止**  

### morphism の性質  
- **一方向性 (non‑reversibility)**  
- **意味を追加しない純粋変換 (pure transform)**  
- viewpoint の構造を壊さずに carrier に写す  

### morphism が持たないもの  
- 意味  
- 状態  
- 編集機能  

------------------------------------

## 🩻 carrier  
**viewpoint に投影された像を受け取るだけのオブジェクト。**  
写真で言えば「印画紙」、レコードで言えば「盤面」。

### 役割  
- viewpoint の出力を受け取る **意味ゼロの受像面**  
- 内部ロジックを持たないことで **混濁を構造的に防止**  

### carrier が持つもの  
- **名札 (identity)**  
  - morphism の接続先を識別するために必要  
  - ただし意味は持たない  

### carrier が持たないもの  
- 意味  
- 変数  
- 内部状態  
- ロジック  

------------------------------------

# 🧩 まとめ  
pre‑MVP の段階では、  
**意味構造を一切持たない純粋な OS カーネルとしての 3 層**  
だけを扱う。

- viewpoint：意味の源泉  
- morphism：一方向の純粋変換  
- carrier：意味ゼロの受像面  

この 3 層は、後に viewpoint が増え、  
morphism が多様化し、  
carrier が動的に生成されても、  
**決して崩れない VOD の基盤** となる。
